Maven的简单使用

发表于2020-05-15|更新于2020-06-18|Maven
|字数总计:1.2k|阅读时长:4分钟

简介

Maven 翻译为”专家”、”内行”，是 Apache 下的一个纯 Java 开发的开源项目。基于项目对象模型（缩写：POM）概念，Maven利用一个中央信息片断能管理一个项目的构建、报告和文档等步骤。

Maven 是一个项目管理工具，可以对 Java 项目进行构建、依赖管理。

Maven 也可被用于构建和管理各种项目，例如 C#，Ruby，Scala 和其他语言编写的项目。Maven 曾是 Jakarta 项目的子项目，现为由 Apache 软件基金会主持的独立 Apache 项目。

Maven环境配置要求

Maven 是一个基于 Java 的工具，所以要做的第一件事情就是安装 JDK。

如果你还未安装 JDK，可以参考我们的 Java 开发环境配置。

系统要求

项目 要求
JDK Maven 3.3 要求 JDK 1.7 或以上 Maven 3.2 要求 JDK 1.6 或以上 Maven 3.0/3.1 要求 JDK 1.5 或以上
内存 没有最低要求
磁盘 Maven 自身安装需要大约 10 MB 空间。除此之外，额外的磁盘空间将用于你的本地 Maven 仓库。你本地仓库的大小取决于使用情况，但预期至少 500 MB
操作系统 没有最低要求

检查 Java 安装

操作系统 任务 命令
Windows 打开命令控制台 c:\> java -version
Linux 打开命令终端 # java -version
Mac 打开终端 $ java -version

Maven的下载和安装

最新版本安装

Maven 下载地址：http://maven.apache.org/download.cgi

下滑页面选择bin.zip的压缩文件（windows版本）点击就会弹出连接进行下载。

下载成功之后解压即可

先前版本安装

若想下载先前的版本点击archves

以3.5.2为例

选择进入binaries目录

windows选择bin.zip的压缩文件

点击链接之后进行下载。下载了之后解压即可。

配置环境变量

前提要配置好java的环境变量，配置JAVA_HOME

将maven选择路径解压安装之后。我们来查看一下maven的当前路径

这就是maven的全部文件

接下来配置环境变量。首先有键点击我的电脑,选择属性选择高级系统设置

点击环境变量

点击新建

变量名为MAVEN_HOME,变量值为Maven的安装路径。点击确定进行添加

滑动鼠标选择变量Path,点击编辑

点击新建,输入%MAVEN_HOME%\bin。点击确定

之后原路点击确定返回。之后按下win+R输入cmd。输入mvn -v查看maven的版本

可以看到出现了我们的Maven的版本信息。恭喜你成功配置了Maven的环境变量

Maven的相关配置

配置Maven的本地仓库

1.首先我们新建一个文件夹。我在于maven安装目录的同级下新建了一个repository目录。作为maven的本地仓库

2.进入maven的安装目录，点击进入conf目录

在此目录下可以看见settings.xml这就是我们maven的配置文件了

点击进入settings.xml

下滑可以看见<settings>标签

通过<localRepository>标签配置maven的本地仓库。此处路径分隔符应该使用的是/,windows使用的是\

修改后保存即可！今后maven下载的依赖就全在此本地仓库中了

配置Maven的阿里镜像源

总所周知maven的中央仓库是访问十分慢的，而正常使用maven下载依赖从中央仓库下载到本地仓库这个过程是什么缓慢的。因此我们要配置阿里的镜像源从而加速maven下载依赖

1.进入maven的settings.xml的配置文件。浏览到<mirrors>标签

2.通过<mirrors>配置阿里的镜像源。保存后即可

配置Maven使用JDK版本

在profiles标签中指定使用jdk1.8

IDEA使用本地Maven

点击选择Settings

配置好maven的安装目录，配置文件，本地仓库。选择Apply在选择OK

点击ok确认之后就能愉快的在idea中使用maven了！！
